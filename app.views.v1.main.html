Базовые классы: FlaskForm

Форма для страницы входа. Используется в Blueprint MainBlueprint.

Эта форма предназначена для аутентификации пользователей, требуя ввода логина и пароля. Содержит поля: - login: строка для ввода логина. - password: строка для ввода пароля. - submit: кнопка отправки формы.

Валидация: - Логин должен содержать от 4 до 50 символов. - Пароль должен содержать минимум 4 символа.

app.views.v1.main.handler_error module

app.views.v1.main.handler_error.handle_error_for_html_views(error_template: str = 'error.html') → Callable[[Callable[[...], Any]], Callable[[...], Any]][исходный код]

Декоратор для обработки ошибок в представлениях (views), возвращающих HTML.

  • Forbidden (403) → Рендерит error_template

  • InternalServerError (500) → Рендерит error_template

  • Unauthorized (401) → Если передана форма, добавляет ошибку в form.password.errors

  • Другие ошибки → Логируются и рендерится error_template

Параметры:

error_template – Шаблон, который используется для отображения ошибок (по умолчанию error.html).

Результат:

Декорированная функция.

Базовые классы: Protocol

Интерфейс для главного контроллера приложения.

Этот интерфейс определяет методы для проверки активности сессии и аутентификации пользователя. Классы, реализующие этот интерфейс, должны предоставлять функциональность для работы с сессиями и аутентификацией пользователей.

Методы: - auth_user(login: str, password: str, ip_addr: str) -> bool: Выполняет аутентификацию пользователя по логину и паролю. - check_activity_session(ip_addr: str) -> bool: Проверяет активность сессии по IP-адресу.

Главный blueprint для приложения.

Этот класс реализует основную логику маршрутов и обработки запросов. Он регистрирует маршруты для страниц входа, о приложении и основной страницы. Взаимодействует с контроллером для проверки активности сессий и аутентификации пользователей.

Атрибуты:

controller (IMainController): Контроллер, предоставляющий функциональность для проверки сессий и аутентификации.
